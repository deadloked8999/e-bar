════════════════════════════════════════════════════════════════════
   🔄 ПОТОКИ ДАННЫХ
════════════════════════════════════════════════════════════════════

1. ЗАГРУЗКА ДОКУМЕНТА:
   
   Браузер → Frontend (DocumentUploader.tsx)
           → API Client (documents.ts)
           → Backend API (main.py /api/documents/upload)
           → File System (uploads/)
           ← Response с ID документа
           ← Обновление UI

2. ПОЛУЧЕНИЕ СПИСКА:
   
   Браузер → Frontend (DocumentList.tsx)
           → API Client (documents.ts)
           → Backend API (main.py /api/documents)
           ← JSON с массивом документов
           ← Рендеринг списка

3. ВЕРИФИКАЦИЯ:
   
   Браузер → Кнопка "Принять/Отклонить"
           → API Client (documents.ts)
           → Backend API (main.py /api/documents/{id}/verify)
           → Обновление статуса
           ← Response с обновленным документом
           ← Обновление UI

   🚀 ПРОЦЕСС СБОРКИ

BACKEND:
  1. python -m venv venv          (создание окружения)
  2. pip install -r requirements  (установка зависимостей)
  3. uvicorn main:app --reload    (запуск сервера)

FRONTEND:
  1. npm install                  (установка зависимостей)
  2. npm run dev                  (dev сервер)
  3. npm run build                (продакшн сборка)

PRODUCTION:
  1. Backend → Docker контейнер
  2. Frontend → npm build → Nginx
  3. Database → PostgreSQL
  4. Files → S3/MinIO
